## 42.多开几个天猫店，到底有什么问题？
我之前帮一个家具店铺做顾问，他在我去之前，做了 5 个 C 店，一个天猫。他们本来是有个 C 店销售最好，然后，他们觉得天猫是趋势，希望我帮他们操作天猫店。在做之前，我给他们打预防针，说如果我来做天猫店，那么后面会出现的情况是，天猫旗舰店冲起来之后，你的 C 店可能都会受很大影响。


果然，当天猫店做到了细分类目的 top3 之后，他那些 C 店的销售额越来越差，最后都关掉了。但是有一点，就是他的一个天猫店的销售，超过之前 5 个 C 店加起来的 2 倍还多。


我想说的是，卖家如果经营多个同类目店铺，会有影响，那么思考一下，为什么淘宝不愿意让卖家经营多个同类目店铺？


之前我们说过，要想看明白淘宝的规则，首先要从「出发点」角度看，淘宝制定规则的出发点大部分是为了用户体验，那么从用户体验上，一个团队经营多个店铺有什么不好的地方？


假设，如果我是某个小类目的 top3，这时候以我的经验，团队，资源等等，我开一个新的店铺，卖同样的东西，肯定要比新手卖家开新店铺容易得多。


如果这样可以，那我只要想提高销售额，最简单的方法就是多开店铺，理论上讲，如果我无限新开店铺，就能做到小类目的垄断。


如果这样做可以见效，那么就会对用户带来这样的不好体验：在这个类目只能看到单一品牌的单一商品，整个平台商品因为垄断变得千篇一律。


我们前面在讲淘宝搜索的工作原理时，提到打散原则，同一个关键词只展现一个店铺的两个宝贝。可能在听到这个规则的时候，有的同学心想，那么我要想在某个细分类目有更多的宝贝能拿到流量，是不是开更多的店铺就可以了？


答案是不可以。


阿里巴巴有个「同人店铺」的判定规则，会把一个团队经营的多个店铺关联在一起。


如果一个团队，经营的多店是同个品牌，同个类目的同类商品，那么对这些关联店铺会带来的影响是，只有一个店铺的搜索展现正常，其他店铺会基于这个表现最好店铺的影响，搜索展现相对比较弱。


所以我们经常会看到，一个团队经营多个店铺，往往只有一个店铺做得比较好，就算你发力做第二个店铺，可能会出现的结果是，其他店铺只要起来了，反而会影响原来销售比较好的店。


需要说明的是，同人店铺的影响，在淘宝和天猫之前也有影响，所以我们会看到有些行业卖家，C 店做得很好，但是天猫很一般；还有，同人店铺的影响，只影响店铺的搜索展现，对于你报名活动，获取官方资源，没有任何影响。但是多数行业，只要搜索流量起不来，店铺就很难起来。


「同人店铺」识别，对于多店搜索流量的影响：


如果是同个品牌，同个类目，相同产品，影响最大；


同个品牌，同个类目，不同产品，影响次之；


不同品牌，同个类目，不同商品，影响更弱；


多个店铺，不同类目，几乎不受任何影响。


淘宝对于「同人店铺」的识别方法，其实有点类似我们之前说的作弊稽查系统，会进行多维度的关联识别，比如多店铺登陆 ip、后台注册信息、法人信息、电话号码、手机硬件，ERP 抓取订单匹配的商品编码、图片空间、发货地址、退货地址等等。


如果你只是其中一个维度异常，比如在某个货源的集散地，几个淘宝店主共用一条网线，但是其他维度都没有串联，那么可能根本就不会受到「同人店铺」的影响；但是如果你有几个维度同时串联，那么这时候店铺就会关联到一起，受到同人店铺的搜索流量打散。


因此，经常有卖家问我，是不是我多店经营，要把网线分开。我一般会告诉他，不用去花这个冤枉钱再弄个网线，因为你可能再拉一根网线也无济于事，原因是你就算网线分开，但只要在「同人店铺」的多维度识别当中有其他的多维度串联，还是会被判定同人店铺。


还有卖家问我，如果一旦被判定多店铺同人后，会影响多久？


答案是，并不会很久，原因是只要规则就会有偶然和误判。曾经有我的朋友和我说，他们尝试摆脱同人店铺成功了，只要把团队和仓储发货完全分开，大概一个周后流量就开始正常了。


但是，问题有两个：


第一：你目前并不能完全弄清楚，同人店铺识别有哪些维度，我自己哪些维度异常，所以你会不断被判定同人。


第二：如果你为了摆脱同人店铺判定，把运营团队和仓储发货都完全分开，对公司来讲，成本大大增加，这样做真的值得吗？


我也一直在和淘宝搜索的小二沟通这个同人识别规则，从商业的角度看，很多品牌在成长之后，都会有切分到细分人群的子品牌战略，所以会有同个团队在同个类目，开多个店铺的情况。


但是，目前如果基于同人店铺的识别，影响搜索流量的话，其实不太合理。


所以，目前基于我的反馈和市场的变化，搜索也会不断的升级和调整算法，让规则更适合市场发展。


如果非要让我给大家个建议，我想说的是，建议大家先集中发力在一个店铺做运营，目前来讲，至少在我们身边，很少有多个同类目店铺都做的比较好的成功案例。
