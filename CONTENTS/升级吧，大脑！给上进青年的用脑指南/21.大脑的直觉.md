## 21.大脑的直觉
你好，欢迎来到知乎大学，我是葛鉴桥。


在之前的课程中，我们讲过许多跟大脑感知觉、还有认知功能相关的内容。而在这一次课上，我想跟大家讨论一下，我们大脑很重要的一种后台活动，那就是大脑的直觉。


其实在我们的日常生活中，很多想法和决定都是直觉式的，往往是你忽然就获得了某种认识、或者忽然就觉得应该怎样去做，而且你往往不会注意到整个决定的过程细节，觉得这个东西一下子就那么来了，我们所说的直觉 ( intuition )，一般就是这样的体验。


用心理学家或者脑科学家的话来描述直觉，一般会有两个关键点：第一点，是对复杂信息以及复杂情境看似毫不费力的处理和评估；第二点，就是在这个决策或者决定的过程中，你的意识几乎不怎么参与其中，也就是说在你意识不到的时候，大脑已经替你做出｢是｣或者｢否｣的选择了。


我一直都觉得，大脑的直觉是一个特别酷、而且特别方便的东西，它能让人省去很多要进行心理斗争并且来回纠结的过程。根据 2002 年诺贝尔经济学奖得主、普林斯顿大学心理学教授丹尼尔卡纳曼的说法，直觉是我们认知加工的两种基本模式之一，是一种并行式、自动、联合式以及更情绪式的加工过程，可以对复杂信息进行快速不费力地评估。也就是说，直觉的存在，就是让我们大脑能特别快速、有效率的做事情，而且，很多时候，依靠直觉，你还可以把事情做得特别靠谱。


我们每个人都有过｢直觉时刻｣，比如，当你听到老虎的咆哮，直觉地就想撒腿快跑；当你听到小奶猫的叫声，直觉地又想上去撸两把；而当你看到一张陌生人的脸，有时候你立刻就能感觉到这个人很值得信赖、或者很不可靠，但你又说不出来为什么。


有些人可能觉得，看脸这个太过玄学，其实除了看脸的直觉之外，还有很多人在许许多多的领域都有很好的直觉，尤其是如果你在这个领域呆了很长时间、接受了大量的信息、进行过加工判断以及训练之后，你就可以对更复杂的信息模式产生直觉。


举个例子，在数学这个领域，我就见过不少有着很棒直觉的人。比如，一个往往需要在草稿纸上演算半天才能算出来的结果，有些数学高手看了一眼，就能感觉出你算的这个结果对不对。我虽然不是数学高手，但是也有幸体会过一次这样的直觉感受。


当时我还在念心理学的研究生，有一次课上讨论一篇最新的认知神经科学方面的方法学文献，里面涉及到一些数学方面的公式，其中有一个 2x2 还是 3x3 的矩阵，矩阵的元素里面还有未知数 x 之类的，论文里把这个矩阵给出来之后，后面紧接着就给出了对这个矩阵求得的逆矩阵。


我还记得当时看着那个逆矩阵，就感觉不太对劲。一般而言，我其实是个很懒的人，线性代数都是好久以前学的了，而且这篇文章还发表在一个很高端的期刊上，作者也是一位颇有建树的资深研究者，这样发表出来的论文怎么会在这种计算上出问题呢。但是当时那个不太对劲的直觉一直挥之不去，我就当场用纸笔算了一遍，发现结果和论文上不太一样。为了保险起见，下课之后我又回实验室用 matlab 重新验算了一遍，发现论文上那个逆矩阵确实是错的。后来我还给作者发了邮件，告诉他这个错误。所以直觉有时候真的是很奇妙的事情。


许多人都对直觉很感兴趣，想知道我们大脑的直觉到底是怎么来的。脑科学家当然也不例外。只不过我们可以产生直觉的领域实在是太多了，根据信息的类别还有具体组织形式的不同，大脑直觉的运作也并不是都一样的。


比如说，我们对语言就会有直觉感受，举个例子，你一定有过这样的经历，话到嘴边突然忘词了，明明自己知道自己要说什么东西，但是就是说不出来；还有的时候，直觉可能就是某种语感。


2007 年的时候，一群德国的脑科学家就研究了这种类似于｢语感｣或者说｢语言直觉｣的东西。他们好奇的是，我们的大脑在语言加工中，对词语意思的直觉是如何产生的，还有这种说不出来的、对语义的直觉判断，和那种我们能明白地说出来的外显判断之间，在大脑加工里有什么不同。他们用了一种叫做｢语义连贯性判断｣的方法。在实验中，受试者会看三个单词，有时候这三个单词在某种程度上是有关联的，比如像，玩 ( playing )、计分 ( credit )、报告 ( report )，这三个词的背后都和｢纸牌｣这个关键词有关系；有时候他们选的三个单词又完全不相关。而受试者在看到这三个词之后，要首先判断这些词之间是不是有连贯性，如果是有连贯性的，到底是和什么词有关联。


有时候受试者可以很清晰地指出，这些词是和哪个关键词有关联，而有些时候，人们会觉得明明这些词之间有着某种连贯性，但是又说不出具体是和什么有关联。后面的这一种情况，反应出来的就是我们语言上的某种直觉了。


所以研究者感兴趣的，就是人们在看到词语之后，能说出具体关联点时的大脑活动，以及明明觉得有关联又说不出到底是什么、直觉式的大脑活动之间，有什么相似和不同的地方。


结果他们发现，在直觉式的语言判断中，相比于外显的判断，大脑的左右两侧下顶叶以及右侧颞上回的区域，都有着更强的神经活动。我们的大脑顶叶往往又被称作是｢联合皮层｣，这个区域被认为是整合不同模态信息的地方；而大脑的右半球，在我们第一模块的课程中
也提到过，被认为很擅长进行更加整体性的加工。右侧颞上回在以往的研究中，也总被发现，在语言加工中扮演整合那种看起来互相之间联系有点儿远的信息。


因此他们认为，这个大脑双侧顶叶和右侧颞上回参与的结果，反映出大脑的语言直觉，就是激活了一系列和语义信息以及语言加工有关的广泛脑网络，把貌似不那么相关的语言信息和语言知识，在后台、不需要意识到地、就整合到一起去，为我们形成了对语言意思连贯性的直觉感受。


有些人可能还会好奇，像语言文字这种我们人类已经用了至少几千年的东西，大脑的直觉是一种情况；那么对新产生的事物，我们大脑的直觉又是怎么回事呢？


2018 年的时候，有一群来自葡萄牙的脑科学家发表了论文，他们想研究的是计算机程序员，也就是码农们，对编程程序里的 bug ——也就是程序错误和漏洞——的直觉，是怎么在大脑里产生的。他们好奇的就是计算机程序以及编程语言，这种对几千年的人脑来说都很新鲜的直觉是怎么来的。


于是他们找来了 20 名职业程序员，平均年龄 28 岁，所有人都有计算机硕士或者博士学位，而且至少有 3 年以上的 C 语言编程经验。在程序中检查代码或者捉虫，都是他们日常工作的一部分。然后他们一边给这些程序员看好几大段程序代码、要求他们找出代码错误，一边扫描他们的大脑。当然，在实验之前，他们会给程序员们说明程序的目的、主要设计思路还有逻辑，在每段代码前也有用自然语言描述的代码说明，为了让程序员们能够尽可能掌握除虫所需的背景信息。然后等看到代码的时候，研究者要求程序员们一旦感觉到某行代码有问题，就立刻按一个｢怀疑｣键标识出来，然后他们可以慢慢再研究代码，去确定这行代码是不是真的有问题。等到他们研究出来，确定有问题之后，再按另一个｢确认｣键来表示。


研究者认为，程序员在迅速产生怀疑时所反映出的，就是编程直觉；而后面的确认过程就不是直觉式的了，而是有意识的、精细思考的过程。


他们考察了大脑活动之后发现，和后面的精细确认 bug 相比，程序员们的大脑在直觉感到有 bug 时，在左右两侧的前脑岛区域有着更强的活动，而且在左侧脑岛的活动强度还可以成功预测之后真正发现有 bug 的准确性。换句话说，那些在直觉感到有 bug 时、前脑岛活动越强的程序员，后面越会发现他们的直觉是对的，果然这就是个 bug，所以说他们其实对程序 bug 的直觉准确性越高。同时他们还发现，大脑的其他重要网络和相关脑区，比如默认网络、顶叶区域、前扣带回等等，在除虫的过程中对前脑岛区域是有着自上而下的调控的。


研究者们认为，脑岛本身就是一个整合语言信息、数学信息等复杂信息的决策脑区，虽然编程是人脑在近几十年才遇到的新情况和新信息，但是大脑把自己已有的｢语言脑网络｣｢数学脑网络｣｢决策脑网络｣｢错误探测脑网络｣｢符号处理网络｣等等重新组织、整合并且再利用起来，共同服务于这个新出现的事物——也就是正确编写程序代码。并且在此过程中，对人脑来说，像编程直觉这种需要利用到这么多复杂信息的新直觉的产生，和脑岛这个最擅长在不确定情境下整合信息、并作出直觉决策的脑区，有着密不可分的关系。


总结一下，今天我们主要介绍了关于大脑直觉的一些最基本的看法，以及一些有趣的脑科学发现。我觉得其中最重要的是，我们必须把直觉当回事儿来看。直觉并不是那么虚无缥缈、玄而又玄的东西，它其实就是我们大脑处理信息、尤其是处理复杂信息的一种方式，和我们所有有意识的思考和考量都是一样的。


或许这个比喻会让大家比较有感觉，有时候我们都会遇到这样的情况，你做了一件其实有点儿复杂的事情，比如你用几十种技法、复杂的构图、深沉的思想底蕴和前卫的艺术理念画了一张画，结果来了一个艺术小白问你，你是怎么画出来的？这时候你会有种千言万语都不知道从何说起、一句两句跟这个人都说不清楚的感觉。很多时候，我们的意识思维就像是这个小白，只能听懂那种简洁明了的话，而大脑在对复杂信息做出加工和判断之后，还要去和这个小白解释自己是怎么做的吗？我要是大脑，我也不乐意费这个劲啊。所以干脆这件事儿具体怎么做的就别说了，你也只能和小白说，就这么画出来了，好看吧？小白点点头说，我很喜欢。我想有时候我们大脑的下意识后台活动和有意识的加工活动之间，大概就是这样咯。


所以，不用否认直觉、它也是你大脑的产物，是我们所谓大脑｢理性、精密思考｣的重要补充，它有时候需要调动的大脑资源以及脑区、脑网络，可能比你进行专门的理性思考还要更复杂，但它就像一个默默为你保驾护航的保镖，只是行胜于言罢了。


好了，直觉告诉我，这节课到这里可以先告一段落了。


咱们下次课再见啦~
