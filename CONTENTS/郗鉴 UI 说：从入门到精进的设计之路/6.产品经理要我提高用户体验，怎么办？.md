## 6.产品经理要我提高用户体验，怎么办？
你好，请坐。这里是郗鉴 UI 说，欢迎你回来，我是郗鉴。


今天要跟大家来聊的话题是产品经理，叫我们提高用户体验时，我们应该怎么办？


很多朋友会觉得用户体验不应该是我考虑的事情，我是设计师对吧，我应该负责把界面做好看了就行了，对吧，用户体验与我何干。其实用户体验这个词，user experience，它不光是我们所想象的那样，就是产品经理或者是交互设计师来完成的。用户体验其实是一种精神，是应该整体的团队都一起来朝用户体验良好的目标去推进的。


我们知道很多的公司都会设置 UED 或者是 UCD 团队，这个是什么意思呢？UED 就是 user experience design 在用户体验设计。UCD 是 user-center design，就是以用户为中心的。这个设计设计部门以这个词作为它的一个名称，代表什么？代表这个团队都应该重视用户体验，应该一起努力推动用户体验。这个团队一般是指是设计师吗？不对，我们在一些招聘的网站上也会看到某某网站的，UED 其实是有设视觉设计师，也就是我们对吧？或者是 UI 设计师，还有交互设计师 UX 或者叫 UE，还有产品经理，对吧？然后整个包括前端在一起的团队都被叫做用户体验的设计部门，通常这个部门是由一个用户体验设计部门的总监来管理的，我们作为其中一环，我们应该与其它部门一起协调，来共同推进我们产品的用户体验度。


也就是说视觉设计师其实确实是应该推动一下用户体验的，对吧？这个用户体验，我们老说用户体验其实有一个理论方面的词叫「用户体验五要素」，它其实是一本书里写的，但是我简单可以跟大家来介绍一下，用户体验五要素是个什么样的一个概念。


用户体验五要素分为战略层、范围层、结构层、框架层，表现层。


这个战略层是说，这个产品的目标是什么，它解决怎样的一个用户的需求。比如说很早之前微信刚刚出来的时候，他的解决的用户的需求，就是用户可以使用微信来与自己通讯录里的好友进行联系，对吧？这是它的一个战略层，战略层其实通常是由这个 leader 来负责的，对吧？就是领导大 boss 来想战略层方面的一个规划，我们朝哪个目标去走？ 


下面是范围层，范围层其实就是我们这个产品，比如说微信它应该有什么样的功能，它应该有什么样的内容，这是范围层，那功能其实并不是越多越好，我们都知道。在我的屋子里也有一把瑞士军刀，瑞士军刀其实有的时候还是挺没用的，对吧？我们做产品的时候千万不能做成瑞士军刀，我们一定要把我们的功能围绕着我们的一个核心的战略来进行的。比如说微信，其实他周围的功能，比如说咱们平时看的附近的人漂流瓶，对吧？朋友圈这些都是围绕着社交来打打造的，对吧？他筛选了很多与自己核心功能相关的一些功能来进行布局，所以范围层是筛选，我们产品需要怎样的功能呢？


下面是结构层。结构层其实相对刚才的这两个层面而言，其实结构层更重要的是一个信息的架构，对吧？整个我们页面之间的逻辑跳转，然后这个页面之间是怎样切换的，都是结构层去构思的事情，范围层、结构层，其实都应该是产品经理来主要跟进的。结构层其实也可以由交互设计师来完成，结构层主要就是要解决页面之间跳转逻辑的事情。说的可能比较抽象，大家可以想象一下我们有的时候是不是进入一些 APP 以后返回消耗很多体力，对吧？


你点进去一个页面以后，你狂按上面返回得按十几下才能到达首页，这是一个信息的层级的一个问题，对吧？还有的时候我们进入 APP 本来是想简单地进行一个操作，但是我找了十几个页面，也没有因为它埋得太深了，所以这样的问题都是结构层上的问题，对吧？它会导致我们用户体验觉得这个是不舒服的，所以在战略层、范围层、结构层上产品经理以及我们大 boss 、大 leader，在围绕着用户体验打造整个产品的时候，就必须要进行用户研究，对吧？分析用户的痛点然后去完成。 


再下面就是框架层，什么是框架层？框架层其实与我们就有很多的关系了，框架层是说我们比如说页面它分配了三个功能，对吧？这三个功能谁是重点？我们不可能让用户去猜，对吧？还有一本书名字，大家应该记住，Don't make me think，不要让我思考。是吧？这本书虽然已经发行了很多年了，但是他这个标题一直在我脑子里环绕，对吧？这是我们做设计的时候不要让用户猜，你会经常看到，如果有两个选项的话，APP 的设计师一定会强化一个、突出一个、然后淡化另一个，就是说突出给用户它主要应该在本页面中完成的功能。 


所以框架层在这方面就需要我们设计师来参与、一起思考了。当然参与框架层的一个设计，除了我们之外，还可以是产品经理，还有交互设计师等等这些人，我们一起来完成框架层的一个设计。很多朋友可能觉得我们是设计师，是不是应该他们给我的原型图就是一个没有上色的感觉，对吧？我把它细化就行了，你就错了，我们在用户体验流程之中要把框架层也要参与进去，我们也要一起去构思这个功能放在这个页面是否合适，如果不合适我们是不是应该把它淡化，或者把它层级加深，把它收起藏起来，对吧？所以框架层也需要 UI 设计师的一个参与，最后的一个层次才是表现层，表现层我相信大家应该都熟了。 


表现层就是我们 UI 设计师本事本职工作了，视觉设计对吧？这个按钮到底长的是红色还是蓝色，这个文字应该是四十几号字还是三十几号字，对吧？那就是由我们来把控了，所以产品、一个产品的用户体验的五要素就是这样，战略层、范围层、结构层、框架层、表现层。


而我们不应该仅仅的去思考表现层，应该去多去思考框架层，以及结构层等等的这些是否做到最优方案，是不是最优的，然后也要去理解战略层跟范围层，因为战略层其实我们作为视觉设计师是无法左右的，对吧？我不能说咱们公司别做社交，咱们做一个金融产产品，对吧？不可能。


范围层其实所选的功能，我们也不应该去做太多的浪费时间的事儿，对吧？我们也不能思考说能不能给 APP 加一个功能，对吧？我们加一个什么样功能，其实这个也不是我们的本职工作，但是我们要理解他们的战略层跟范围层，当然也可以适时给出意见，那也是很好的，但是我们无法决定这两个层面。但我们可以去帮助产品经理以及交互设计师，我们一同去思考结构层、框架层、跟表现层的东西。所以说 UI 设计师或者我们视觉设计师确实是用户体验设计中的很重要的一环，而且并不仅仅是表现层才由我们来决定，我们也要去一起去参与其他几个层面上的一些思考。


所以这个就叫做用户体验的五要素了，大概知道这一个结构上的东西的话，我们具体怎么来推进这个设计，我们怎么才能够让用户体验的在视觉层面上感觉到用户觉得非常爽。除了我刚才说的一些功能的摆放，对吧？一些功能是否要突出之外，我们还可以做一件事情，这个事情就叫做情感化设计。情感化设计其实最早是设计心理学中的一本书的名字，设计心理学是这个唐纳德博士来写的，他是认知心理学的一个专家。


情感化设计具体到我们 APP 中，它有什么样的一个例子给大家来讲一讲，比如说我们在使用一些定外卖的软件的时候，如果是下雨天，你是不是更着急了，对吧？你也很这时你想催一催，外卖小哥。打开饿了没对吧？发现饿了没里面也在下雨，而且还在打雷，对吧？外卖小哥在路上就是已经满头大汗骑这个车，你觉得好像有点心疼，因为你看到这个图形跟你当时的一个想法就已经不一样了，对吧？然后催单还隐藏在更多里，对吧？它不是突出的，然后它隐藏在这个更多里，所以你其实就会通过一些设计来缓解掉你焦虑的情绪了。对不对？你也知道他小哥也不容易，他正在哪个地方正在加紧过来，手机里也乌云密布，对吧？


同时还有比如说，比如说还有一个软件，还有一个手机的 APP 叫 boo！。这个 boo ！是一个小学生的社交软件，小学生现在用手机智能手机的人非常之多，报的情感化设计做得非常好。他怎样做的？他是比如说你登录的时候，输入用户名的时候，身边有一个非常可爱的独眼的一个小怪物在看着你，是吧？在朝你笑。你点密码的时候他会遮眼睛，对吧？然后你进入了以后，他会跟你说嗨是吧？然后给你介绍一下我们这软件的一个功能，所以说这样的设计会把我们本来无聊的操作变得非常有趣，对吧？


同时情感化设计也是一些对于用户效率、用户乐趣的一些思考，有一个让我比较印象很深的一个设计，就是 Gmail，就是谷歌的邮箱，谷歌的邮箱在登录的时候，它有一个特别的设计。我相信大家在登录用户名跟密码的时候都会遇到，密码输入错误的问题，对吧？我曾经（咱们就不说是哪家银行了），在一个银行的客户端里，我要进行一个对公的一个操作。我输错了三遍，密码也很正常，我输错一遍密码就密码错误，然后密码错误，我在是第三个的时候，弹出一个密码已被锁定。然后我说那怎么办？我该解锁。


在网站找了半小时以后，我找到一条新闻，他用新闻的形式告诉我，说这个密码如果被锁定，你必须拿着身份证去开户行解锁。所以其实你一开始应该警告我，对吧？你密码输错了以后你可能把我锁定了，他也没有警告我对吧？他就是一直告诉我错误，他不告诉我哪错了，其实我都不知道，不太知道我到底是大小写输错了，还是我这用户名本来就写错了。


谷歌的机卖他在登陆的时候有个什么样的独特的设计，有一次呢我登陆的时候，它觉得我不太安全，他让我改变密码，然后后来登录的时候，我输的是之前的密码。他跟我谈提示的时候他这么说，他说你在一个月前更换过密码。然后当你大小写的时候，不是不容易输错吗？它会告诉你大小写是开启还是关闭状态，对吧？如果你输错了这用户名的话不存在的话，它也会去服务器查询用户名是否存在，然后再告诉你，所以这样的话其实就解决了一个用户就很萌的状况，对吧？


我登录错误了，我哪错误了，你要告诉我对不对？所以这是也是一个情感化的设计，所以情感化的设计其实就是我们可以理解为我们用一些视觉以及交互的方式让用户使用产品的时候更爽，对吧？同时我们也可以调用用户的情感来激发他做某一些行为的可能性。


情感化设计分为三个水平，第一个水平叫做本能水平，本能水平的设计指的是像我刚才说过这样一些产品的用户体验设计，他是非常直观的让你直接能感受到的它一个画面的本能水平。比如说我们在 618 或者双 11 、双 12 的时候，各大电商的产品都会出一些这种运营的产品，你点击它以后，就会感觉到非常的有视觉震撼的感觉，然后感觉这东西特别便宜，你就想赶紧买，对吧？这其实就是它商家用的情感化设计的一个方式，来推荐你来购买它的产品。我们都知道 618 或者是双 11 、双 12，它们的色调都是非常明亮非常刺激的，对吧？它不可能做成我们老说这种什么性冷淡对吧？现代主义设计风格，它不可能做成这种风格，他一定做的是非常有让你热血喷张的感觉。


它会用一个头图，这头图一定是非常立体的，因为立体的东西能抓人眼球，然后用紫色或者是红色橙色，对吧？这些刺激我们视觉神经的颜色，来完成整个页面的设计，所以当我们看到的时候，我们就非常觉得很激动，对吧？


同理还有各大 APP 在春节、情人节、七夕这些节日的时候，他们的 icon 以及他们的闪屏也会做相应的情感化设计，对吧？但是情感化设计仅限于视觉刺激本能水平这个方面。对吧？比如说我们知道过年的时候打开手机以后，大家都是红 icon 了，所以这是本能水平的情感化设计，它也会给人一种非常舒适的感觉，或者非常激动的感觉。


第二个水平就比本能水平稍微高一个级别叫行为水平。什么是行为水平？行为水平就是我们在使用产品的时候，这个产品能够让我们更有效率，而且在体验的时候有乐趣。比如说我们在玩微信的时候，有一个叫附近的人，对吧？然后还有一个是摇一摇，对吧？还有漂流瓶，这都属于陌生人社交。摇一摇，你摇的时候会有什么样的一个声音，它是一个猎枪的声音，对吧？我们都知道找陌生人聊天的时候，你肯定是有一种这种猎奇的心理，所以它用的是一个猎枪的音效，而且他本身这个操作起来是有挺有意思的，它是要摇手机摇一摇，对吧？后来有摇一摇支付对吧？有电视可以摇一摇，然后可以收到一样的资讯，所以说摇摇这样的设计其实就属于行为水平的情感化设计了。


最高的一个级别的情感化设计叫做反思水平的情感化设计。什么是反思水平的情感化设计呢？它让你反思吗？当然不是了，反思是一个它的名称，其实反思级别的情感化设计会让用户对我们的产品产生依赖感，甚至产生一个粉丝的效应。它会认为这个产品是属于他的，他会觉得有归属感，这就非常的厉害了，不是每个产品都可以做到这个级别的。


比如说我们知道大家现在很喜欢非常可爱的一个熊跟一个兔子他们俩在谈恋爱，对吧？还有一个小小小鸭子有点像周冬雨，对吧？还有小青蛙什么的，它们是一组的，对吧？他们这组人叫做 Line friends。Line 是一款社交软件，它是韩国公司 Naver 旗下的一款社交软件，有点像我们腾讯旗下的微信，这个 Line 在东南亚地区非常的火爆。它这个火的原因并不是它功能有多么的出色，因为满足它功能的有微信，还有国外的 What's App。为什么用户对它会有一个忠诚度？是因为他的设计以及它卡通形象，对吧？深入人心，人们觉得这个贴纸实在是太猛了，对吧？非常喜欢他。


同时还有的有一个例子，我们经常会看到很多很多我们腾讯的员工喜欢公仔，对吧？企鹅这公仔 QQ 的形象也深入人心了，对吧？我们一看到这 QQ 就想到这个企鹅，就想到 QQ 了，对不对？所以这也是一个情感化的设计，它本身的形象带入到了用户的一个心理，对。所以说我们会看到越来越多的 APP 或者越来越多的大公司都希望自己能有一个品牌感，能有一个卡通形象，甚至是对吧？就是为了让情感化设计能够拉近我们产品与用户的一个距离。


所以如果产品经理再让我们去提高用户体验的话，我们就应该知道该怎么做了吧。首先用户体验的五要素来说，我们应该在原型图工作的时候我们就参与进去，我们也来发表我们的意见，比如说这个按钮是不是应该放在这里，功能应不应该露出，用功能应不应该淡化？然后我们在做视觉设计的时候，可以去更深入地就把自己想象成一个用户，在使用产品的时候会不会有一些情感上的变化，一些焦虑的情绪，或者是一些需要我们设计去暗示的情绪，对吧？


比如说我们如果是电商的话，是不是应该用这种非常明快的颜色来刺激用户消费，然后如果我们是一款社交产品的话，我们是不是应该去设计一些让人一眼就能够识别出来的卡通形象，作为我们这个产品的一个代言人，对吧？我们在做设计的时候，是不是应该设计一些很多的错误的提示，让用户知道到底哪出错了，并且告诉他应该当下应该怎么做，不要让用户迷茫，也不要让用户思考。
