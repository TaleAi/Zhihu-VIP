## 49.【加餐】林纳斯：为乐趣工作一辈子
【加餐】林纳斯：为乐趣工作一辈子


如果你在用安卓系统的手机，你该感谢一个叫林纳斯的人。如果你上网、在淘宝上买买买，你也该感谢他。林纳斯，被称为是 Linux（编辑说明：注意读音）操作系统之父，安卓手机是基于它开发。


林纳斯这个人，背后没有一家庞大的公司，在过去几十年时间里，他就在自己家的一间房间里编程写代码。1991 年，他在大学宿舍里写了一个玩具一样的操作系统 Linux。现在，这个开源的操作系统是互联网最重要的基石之一，或许之一都可以去掉。最近他接受了一次采访，在被问到，再过 25 年会怎么样呢？他说：「已经干了快 30 年了，估计到时候还在干吧。」


林纳斯在为乐趣而工作，这几乎是我们想要的梦想工作了吧？在谈到工作时，我们想到的都是团队、公司、薪水，我们普通人很难有林纳斯这样的工作，当然更难达到像他这样的成就。但不管怎样，我们都可以从他的工作方法中学到很多。今天，我来分享，我从他那里学到的四课。


从林纳斯身上，我学到的关于工作的第一课是，造个玩具自己玩，如果你恰好找对了地方、用对了方法，你真的可能改变世界。


1991 年，林纳斯还是芬兰的一个大学生。他学着写个操作系统，没有想到自己编程写的东西会有什么大影响，他就是写个玩具自己玩。


他编程做了个粗糙的东西，就把它放到互联网上，说，这是 0.01 版。过了一个月，他又发布了 0.02 版，0.03 版。当时，大企业和大学有 UNIX 操作系统，公司和个人电脑有微软的操作系统，Linux 只是一个玩具。


林纳斯很幸运，他找对了地方，有很多极客想要自己折腾电脑，并且，这时处在互联网兴起的前夜，大家都想有这样一个可以做网络服务器的操作系统玩一玩。林纳斯也选对方法，他把 Linux 操作系统代码开源出来，并邀请全世界的程序员一起来开发。


到了 1998 年，按林纳斯自己的说法是，他和 Linux 突然之间一夜功成名就。大公司开始对开源的 Linux 感兴趣：开发 JAVA 语言的太阳软件公司加入进来，IBM 加入进来，数据库软件公司 Oracle 加入进来。


比尔·盖茨因为操作系统成为了世界首富，当时的微软公司强大无比，但是，一个学生宿舍里的玩具一样的系统竟然成为微软的对手。更奇妙的是，它根本不是一家公司，而是程序员们业余时间凭着兴趣开发的东西。虽然不是任何一家公司的创始人、CEO，但林纳斯成为财富杂志的封面人物，因为他是世界的英雄。


从林纳斯身上，我学到的关于工作的第二课是，有时候，如果你想创造一个伟大的东西，你得非常「懒」，像林纳斯那样「懒」。


在说起 Linux 的成功时，林纳斯说，这很大程度上应该归功于他的两个缺点：第一，我很懒散。因此，他没有创建一家公司，而是把 Linux 开源出去。第二，能让别人做就让别人做。他管理 Linux 开发的方式说得好听点是「授权」，说得中性点是只抓住核心，其他都不管。


比如，关于他怎么处理电子邮件，有这样一个故事。有人曾经问他，你出门几天，积压了很多电子邮件，你怎么办呢？他说，我直接删了那些没看的邮件呀。他的逻辑是，如果你真有重要事找我，你会再来找我的。关于邮件处理，他有两个规则，这就是第一条：「如果邮件重要，对方会再次发送」。


而他说，另一条规则比这第一条还重要，第二条规则是：如果（一个邮件）不必回复，那就不回复。


他并非功成名就，所以懒散地休息。事实上，在过去 30 年里，他始终是 Linux 操作系统真正的核心开发者，系统核心的东西都是他亲手处理的。他的这种「懒」，让他可以把精力放在真正重要的工作上。


而在他身上学到的工作第三课是，即便你功成名就，你还可以再造个工具给自己用，也给大家用。永远不要停止。


做软件开发，特别是很多人协同的开发，一个关键的工具是代码仓库管理。林纳斯一直在用一些别的工具，但总有很多不爽的地方。


终于有一天，既然需要，他决定自己写一个适合自己的工具，这就是 GIT（读 G-I-T）。他的动作真的非常快，2005 年 4 月 3 日，他开始开发，到了月底 4 月 29 日，已经基本开发完成。然后，这个工具在 Linux 内核的开发中就用起来了，到了 6 月 16 日，不到两个半月，这个新的工具就正式发布了。


对很多非软件开发领域的人来说，GIT 不算什么。但它非常重要，不用 Linux 操作系统的人也在用它，它同样已经是整个软件行业开发的基石，可能比 Linux 的渗透率还要高。我也一直在用，它真的比之前所有其他的系统都好太多，是真正革命性的工具。


这就是林纳斯，如果觉得工具不好，那就自己造一个。他实在太厉害了，竟然又达到了一个新的顶峰。


对我们普通人来说，造个工具给自己用，同样也非常有意义。这是从「初级的会工作」向「高级的会工作」迈进的关键门槛。


从林纳斯身上，我学到的关于工作的第四课是，如果你想要从工作中获得真正的乐趣，重要的是，你做的工作就是你的目标，工作任务本身能让你自得其乐。


林纳斯曾经出过一本传记，书名是《只是为了好玩》。在他看来，做事的激励因素，按照顺序分别是，一，为了生存，二，为了获得他人认可，三，为了好玩。他很幸运，几乎从一开始就在最高级的第三层次，他做的所有一切都是为了好玩。


如果我们想要更好地工作，我们也该努力进化到这个阶段。这是有着心理学研究支持的，心理学家米哈里提出的「心流」是关于快乐体验的主要理论，他认为，获得极致的快乐体验，最重要一点是，你的目标是要不依赖于外部的薪酬奖金、社交认可、荣誉等等，而应该是「目标不假外求」。它指的是，一个事它本身就是目标，你可以从中自得其乐。比如，我们玩游戏，就是为了玩游戏，这个游戏本身就是目标。


小结一下。我们今天一起看了从 Linux 操作系统创始人林纳斯的工作故事，我们从他那里学到了四课：第一，从小目标开始，造个玩具自己玩；第二，聚焦在自己的核心任务；第三，完善自己的工具，成为高阶的工作者；第四，让工作本身就是你的目标，进入到工作的最高境界。
