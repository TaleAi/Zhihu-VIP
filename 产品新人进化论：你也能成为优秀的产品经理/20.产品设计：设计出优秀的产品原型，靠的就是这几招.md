## 20.产品设计：设计出优秀的产品原型，靠的就是这几招
这一讲我想和你聊一个有关产品原型设计的话题。


都说画产品原型图是产品经理的必备基本功，但由于每个公司对产品的原型要求各不相同，直到现在业界也没有一个统一的标准去衡量原型图的好坏。所以很多新人的原型图会经常被吐槽画的太「粗犷」，或者又过于追求「精美」导致浪费大量时间… 


那有没有什么办法可以画出大家都认可的优秀产品原型图呢？今天，我们就一起聊聊优秀的产品原型图应该怎么画。


先让我来问你个问题，我们为什么要画原型图？


**知识：产品原型图**


**产品原型图也叫线框图，是用线条和图形绘制出的产品框架。它是产品从概念到落地的关键点，也是产品经理跟其他同事沟通的重要工具。**


虽然大家都在画原型，但实际工作中「产品经理、UE 设计师、UI 设计师」的原型侧重点却各不相同。 


**知识：产品经理\UE\UI 眼中的原型**


产品经理眼中的原型是一种粗略的线框图，为的是让大家一下就能看懂产品想表达的意思，便于向他人准确的传递产品需求。产品经理会特别关注原型所表达产品的大体界面、业务流程和产品功能的展现。 


交互设计师眼中的原型则是一种带有各种状态和逻辑规则的精细线框图。目的是快速呈现产品的交互逻辑、便于使用原型进行产品的评审、修改与最终的视觉设计。所以交互设计师更关注原型布局与结构、交互细节、任务流、功能与内容、以及各界面元素的位置/层次/顺序。


我们的 UI 设计师，既不关注产品长什么样，也不关注产品的界面逻辑。他们关注的是产品的视觉呈现。目的是完成产品的视觉设计、诸如产品的情感元素以及控制产品的视觉风格。视觉设计师会更关注原型的整体 UI 设计、色彩与质感、ICON/配图和排版等。 


所以，我们要清晰的了解不同角色在原型图上的分工。不要拿着带密密麻麻交互规则的原型图去找交互设计师，也不要拿着五颜六色的原型图去找视觉设计师。


在正式开始画图之前，我们需要做好准备工作，才能让接下来画原型图的工作起到事半功倍的效果。


**知识：画原型前期准备**


很多新手同学喜欢上来闭门造车，直接就开画产品原型，这样是很不对的。你要知道，产品是拿用户去用的，不同用户对产品的使用方式不同。 


这就要求我们**画原型之前**，花一点时间去**了解**我们的**用户是谁**，**了解**他们的**社会属性、基本属性、使用爱好**，对我们画原型是非常有帮助的。关于这一部分，同学们可以翻看之前用户调研的部分来回顾学习。 


仅仅了解用户是不够的，我们还需要**了解**他们在什么时候使用我们的产品，也就是**用户的使用场景**。比如你的 APP 产品用户大多数都是在公交或地铁上使用，这时候你需要考虑让你的产品原型在用户单手持机下就可以顺利完成，而不是用双手来操作。换句话说，**我们要知道用户使用产品的主场景和扩展场景，在画原型图的时候将场景有效融入进去。**


只了解用户和场景是远远不够的，我们还需要**了解产品帮用户解决了哪些问题以及产品的亮点是什么**，做到需求和亮点于胸中，这样我们画原型的时候才能更聚焦于产品的功能，而不是无约束的画了一堆无用的页面。


**在画原型前**，我们还需要**分析产品的信息结构图、功能结构图和页面结构图**。只有**充分掌握产品页面、信息和功能的上下游关系**，画图的时候才不会迷失，**避免出现页面漏画，缺少页面信息元素和产品功能缺失的问题**。


上面说到的「用户场景分析」和「三种结构图」是你原型文档中不可缺少的要素，也是你向交互设计师传达产品原型时候必要的组成部分。这一部分，我们在之前的课程中都介绍过了，忘记的同学可以回看之前的课程，重新学习一下。 


说了这么多画原型图的准备事项后，终于该说画原型图的流程步骤了。


**知识:产品原型的流程步骤**


在**产品早期阶段**，产品原型往往要经过大量的修改，过早的使用工具反而会大大加重你修改的成本。在这个时期，手绘草图就是你最好用的工具，无论是**在白板或白纸上**都可以**迅速**的**画出草图**。待确定好原型图的基本展现样式后，可以使用工具进行低保真原型设计。


通过手绘草图确定好原型的基本样式后，我们就可以使用工具进行低保真原型的设计了。随后，可以拿原型稿找交互设计师出交互设计稿。最后，拿着交互设计稿找 UI 设计师完成产品高保真 UI 稿的设计。


![img](https://pic4.zhimg.com/v2-1a518bdf7ec5fef41fef0221fd795783.webp)

如果说正确的步骤会帮助你不走弯路，那么下面要说的全局设计则会帮你事半功倍。 


**知识：原型图全局说明**


这里说的的全局说明是指原型图上的统一说明部分。**一般包含统一的页面内交互（如统一弹窗）、规则说明、页面间交互（如全局页面跳转方式）说明和页面异常交互（如无网络、弱网情况）的统一处理说明。如果是移动端需要加上全局可用的手势图示说明**。这样，在接下来的原型设计中，遇到同样的问题不用每次都重复画图并一一说明，只要引用上面的全局说明就好。

到这里，我们的准备工作终于宣告完成，下面我们将正式步入画图阶段。


这里我想强调一下：我的课程里不想讲什么原型设计工具怎么用，事实上，网上实在太多的资料可看了；也不想推荐什么好用的工具，因为无论是 Axure、墨刀还是 sketch，只要能正确的画图，就是好的工具。我要跟你说的是新手在画原型图上常犯的错误和注意事项，因为只有了解了这些知识，你才能尽量避免错误，画出高质量的原型图来。 


**知识:原型设计注意要点**


**1.产品原型不是最终设计稿**


我们必须要知道产品原型不是最终的设计稿，即使你的原型画的再好，无论是交互还是 UI 设计师依然可以根据他们的需求修改你的原型图。所以，在别人改你原型图的时候，请不要拿出跟人「拼命」的态度来阻挡，**产品经理的原型只是一个传达需求的沟通工具，别人每一次的修改都是为你的产品更加完善。**


**2.确保每个细节的正确性**


这个问题不需要多说，**原型设计要确保细节的正确性。**如果产品经理交付的原型细节都是错误的，交互和 UI 输出的结果只会错上加错，工程师按照错误的方式继续开发产品，最后交付给用户一个错误不可用的产品。因此，我们**在做完原型后，需要检查原型的正确性，确保细节都是正确的。** 


**3.必要时添加些文字描述**


我们在画原型时，有时候为了想要准确的表达清楚，就拼命的做各种动效，结果费时又费力，还没法说清楚想表达的具体的内容。这时候，如果我们**在原型旁边加上一些文字描述来说明我们的意图**，这个问题就能很轻松的解决掉。有时候偏偏 5 分钟能解决的事情，我们却要偏偏花费 2 天的工作量，非常的不值得。


![img](https://pic3.zhimg.com/v2-adc8c6c12698991cf0f89f581a614273.webp)

**4.通过明暗描述优先级**


我见过有些产品经理在画原型图的时候在努力的给图上色，弄得整个原型图花花绿绿的，以为这样才够专业。其实无论你多么努力的上色，都没有 UI 设计师做的专业，反而还严重的影响了 UI 设计师的视觉设计工作。


**正确的做法是使用黑白灰色彩的变化去描述原型的优先级，有时为了强调，偶尔可以加一点色彩**，但不要弄得满屏都是。


![img](https://pic3.zhimg.com/v2-38461ae5ca40c1915b708326e5b0f15f.webp)

**5.采用真实的数据填充**


很多产品经理在画原型图的时候喜欢用占位符来代替真实的数据，这是个非常不好的习惯。占位符本身无法模拟真实环境中可能出现的极端情况，特别是在移动端这种界面展现能力不足再提下，如何合理的展现内容就变得尤为重要。所以，大家在画原型的时候，不但要**用真实的数据，还要专挑一些极端的数据展现在原型图上**。这样的原型图才能**合理展示产品最真实的界面**。


原来这个原型图要展现这么多东西啊，是不是我们什么都要画在原型图上呢？答案是否定的，下面这些内容就不需要出现在原型图上。


**知识：原型图上不需要添加的内容**


首先，我们**不需要写视觉规格（间距、尺寸）**这些东西，这是视觉做的事情；其次，我们也**不需要写业务逻辑（这个功能是怎么设计的）**这些东西是在产品 PRD 上呈现的；再其次，我**们还不需要写功能描述（功能的实现机制）**这也是在 PRD 上呈现的；最**后，我们不需要写详细的文案描述，**有时候你需要的是一个原型文档加一个 word 的文案描述。 


既然说到了在原型图上写文案，下面我就要跟你聊聊原型图文案应该怎么去做整理。


**知识：原型图的 wording 整理术**


在原型设计上我们需要学会**用最精准简洁文案来表达**，千万不要像「唐僧」一样啰嗦的没完没了。你要知道，**用户根本没有耐心看完你所有文案，文案写的越多用户的心里抵触感越强。**


比如一个自拍 APP 的产品介绍文案，唐僧会啰嗦的把什么都写上去，而我们要的仅仅是「爱自拍，年轻人的自拍神器」这样简洁有效的文案。 


光精准的表达还不够，我们还**需要用大家听得懂的大白话**。想一下，如用户看到的错误提示文案是「缺少 user token 串、服务器并发处理失败….」会怎样？会不会有抓狂的感觉呢？所以，我们在写原型的文案时，要先把自己变成用户，使用用户能懂的类似「连不上啦，网络不太给力啊」这样的写法去写文案，用户才能听得懂。  


做到以上两点还远远不够，我们还**需要把文案的风格跟用户的类型对齐**。比如我们的用户偏高大上，你就不能写很 low 的屌丝风格文案；再比如你的用户是年轻女性，你的文案就要用像「美美哒，小仙女….」这样的词汇,**尽可能把你的产品气质体现出来。** 


最后，**写好的文案一定要检查一下是否有错别字、语句是否通顺、标点符号是否都用对了**，不要在文案上犯低级错误。
