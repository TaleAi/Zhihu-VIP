## 3.做好项目管理需要哪些常用工具？
项目管理需要哪些常用的工具？我们在执行项目管理的工作期间，会使用到辅助我们完成工作的工具，如之前所提到的，如果想成功地实施项目，需要三大部分的知识，一个是项目管理专业知识，二是一般管理的通用知识，最后需要项目相关应用领域的专业知识，同样的道理，如果我们将项目所需工具进行分类的话，那也是三类，项目管理特有的管理工具，一般管理工具和项目相关应用领域的专业工具。


**一般管理工具**


比如说 HR 部门的考勤系统。比如说财务的的费用审批系统，或者公司的在线 OA 审批系统。


**项目相关专业领域工具**


可能是 IT 项目的编程工具，或者房地产项目的建筑材料成本核算的工具，这些并不是项目的通用性的工具，而是某一个行业专家使用的特殊工具。


**项目通用工具**


在介绍工具之前，先和大家来分享一下项目管理包含哪些工作内容？因为工具一定可以帮助解决工作内容的一部分甚至全部，而如何判断项目管理工具的好坏，其实就是判断工具能多大程度地实现项目管理的需求。


项目管理需要完成以下 14 项内容：


1、对项目进行前期调查、收集整理相关资料，制定初步的项目可行性研究报告，为决策层提供建议。协同配合制定和申报立项报告材料；


2、对项目进行分析和需求策划；


3、对项目的组成部分或模块进行完整系统设计；


4、制定项目目标及项目计划、项目进度表；


5、制定项目执行和控制的基本计划；


6、建立项目管理的信息系统；


7、项目进程控制，配合各项目干系人对项目进行良好的管理；


8、跟踪和分析成本；


9、记录并向产品经理传达项目进展信息；


10、管理项目中的问题、风险和变化；


11、项目团队的组织建设；


12、各部门、各项目组之间的协调并组织项目培训工作；


13、项目及项目经理考核；


14、理解并贯彻公司长期和短期的方针与政策，用以指导公司所有项目的开展。


这 14 项项目管理的工作中，其中有 8 项涉及对外的沟通，至少 10 项涉及汇报，那我们就清楚了，一些办公室的通用工具，是我们不得不掌握和熟练应用的，比如说 PPT，比如说 Excel。


再者，项目组织中建设的沟通渠道，也是需要工具来辅助的，最简单的，有的团队就用微信群做沟通工具，有的小型创业公司用 Teambition，大型的企业用邮件系统，用微软的一整套解决的方案，甚至需要思科的视频会议系统。


除此之外，我们也能看到一些特别的功能需求，是项目管理才需要的，比如制定项目的目标，制定项目计划，安排项目进度，监控项目的进展，控制和分析成本，监控项目的风险和变化等等。


还有一些工作，我们无法定向的找到对应工具，比如说项目团队的组织建设，团队建设可以是一次出游，或是召开一次会议，因为没有一个特定的形式，所以也不属于项目管理工具所要求的。


其实，项目管理工具的优劣就是评估它所具备项目管理的功能的强弱，最基本的项目管理工具，一定要具备的功能包含：


**能定义任务**


**定义项目任务的时间**


**能对任务指派负责人**


**能在工具当中分配资源**


**能对项目的执行情况进行跟踪**


**能建立子项目模块**


**能定义项目的里程碑**


**能建立任务与任务之间的逻辑关系**


**能将任务进行简单的分组**


**具有基本的筛选过滤功能**


那更高级一些的项目管理的工具可能拥有强大的信息管理系统。


嵌入工具内的沟通和交流系统，基于云端的协同办公系统，有强大的数据报表能力，有风险预警和变更的能力，能对任务进行提醒，拥有高级的过滤和筛选功能。


接下来我们横向对比专业的项目管理工具。


项目管理工具也分为，专业工程类的项目管理软件，非工程类的通用项目管理软件和其他的协同办公的工具。


学习这节课的目的是学会选择适合你的工具。


首先，工程类的项目管理软件，是世界上最大的建筑和工程项目管理系统，给大家一个例子，这一类的工程的相关软件，比较多的应用在的大型建筑工程项目，比如说三峡，或者一些大型的房地产项目，一些建筑项目，它就属于这种在特定领域之内起作用的项目管理软件。


如果中国今天修长城，也会应用此类软件，此外，项目管理专家会使用某款非工程类的通用项目管理软件，比较专业，强大的，比如 Primavera 公司出品的 P6，他有两个版本，集团企业级别的 EPPM，公司和团队级别的 PPM，前者适合项目集管理，后者适合单项目管理。


如果是 MAC 系统，你需要了解的是 Omniplan，这是苹果电脑上最专业的项目管理软件，此软件可直接运行 Windows 下的 Microsoft Project 文件，此种文件是基于 Windows 环境由微软公司开发的项目管理工具 Project 生成的文件，该软件也是 Office 套件中的一部分，协助项目经理发展计划、为任务分配资源、跟踪进度、管理预算和分析工作量。


如果你即将成为一名专业级别的项目管理者或者立志于从事大型项目的工作，omniplan 和 Project 是你必须掌握的，但如果你只想了解项目管理的思维，基本的理论学习后能快速落地，我向大家推荐一款思维导图工具——Xmind。


Xmind 虽然不是专业的项目管理工具，但软件中的项目甘特图模块可以帮助大家规划和实施项目、监测其状态、保持对项目预算的控制、并帮助你与团队成员和利益相关者进行有效沟通。


推荐它作为入门级的项目管理工具主要是因为操作简单，上手快，学习成本低，适合入门者去学习项目管理，尤其适合个人项目管理规划或者十几人的小规模团队的项目管理。


而除此之外的，适合中型团队，需要团队密切协作的跨平台的项目管理平台，市场上就比较多了，比如 teambition，明道的项目管理软件，@team，这种平台基于云端技术，可以起到很好的协同办公的作用，但虽然功能很丰富，但在项目资源配置和时间规划维度不够专业，比如，这些工具可以形成甘特图，但不能构建任务与任务间复杂的逻辑关系，即使如此，它仍可以胜任几十人团队，中型规模的项目管理，推荐 teambition。


除此之外还有一些协同办公的工具，这种工具需要和项目管理软件区别开，给大家一个标准，所有协同办公的工具一定是基于云端的，但有些工具连项目管理核心的工具甘特图都没有，也无法生动的表现工作分解的结构和过程，那这种软件，仅仅建设了一个协同办公的空间和平台，并非是真正的项目管理工具，比如石墨文档，slack，这些并不适合做项目管理。


回顾一下，我们分别讲到了项目管理工具与其他工具的区别，项目管理的工作内容，主流的三种类型的项目管理工具，推荐的个人入门级的项目管理工具 Xmind，小型团队协同办公的项目管理工具 Teambition 和专业级别在 MAC 环境下的 Omniplan、在 windows 环境下的 project。
